### 방법 3: Python 코드에서 직접 사용

#### 기본 사용

```python
from preprocessing_examples import quick_preprocess

# 전처리 실행
df = quick_preprocess('input.csv', 'output.csv')

```

#### 최소 길이 조정

```python
from preprocessing_examples import preprocess_job_postings

df = preprocess_job_postings(
    'input.csv',
    'output.csv',
    min_text_length=100,  # 100자 이상만 유지
    verbose=True          # 진행상황 출력
)
```

#### 개별 함수 사용

```python
import pandas as pd
from preprocessing_utils import (
    clean_text,
    remove_duplicates,
    combine_job_fields,
    filter_by_text_length,
    get_job_distribution
)

# 1. 데이터 로드
df = pd.read_csv('input.csv')

# 2. 중복 제거
df = remove_duplicates(df, subset_columns=['회사명', '공고제목', '주요업무'])

# 3. skill_text 생성
df['skill_text'] = df.apply(
    lambda row: combine_job_fields(row, ['주요업무', '자격요건', '우대사항']),
    axis=1
)

# 4. 짧은 텍스트 필터링
df = filter_by_text_length(df, text_column='skill_text', min_length=50)

# 5. 직무별 분포 확인
dist = get_job_distribution(df, '직무')
for job, count in dist.items():
    print(f"{job}: {count}건")
```

## 🎯 논문 적용

### III.2 데이터 수집 및 구성

```
수집 데이터에서 중복 게시, 모집 종료 후 정보 누락, 복수 직무가 혼합된 
공고 등을 점검하였다. 동일 기업, 동일 직무, 동일 본문을 가진 공고는 
1건으로 통합하였고, 직무 텍스트가 지나치게 짧거나(50자 미만) 세분류 
판단에 필요한 설명이 부족한 공고는 제외하였다.

최종 분석 데이터는 중복 제거 및 필터링 과정을 거쳐 총 282건으로 
구성하였으며, 세분류별 분포는 36건(인공지능서비스운영관리)에서 
49건(생성형AI엔지니어링)으로 실제 산업 수요를 반영하고 있다.
```

### III.4 데이터 전처리

```
전처리 과정에서는 특수문자, 중복 공백, 불필요한 줄바꿈을 제거하고, 
주요업무·자격요건·우대사항을 하나의 문자열(skill_text)로 통합하였다. 
Python, PyTorch, LLM, RAG와 같은 기술 용어는 의미 손실을 막기 위해 
원형을 유지하였고, 단순 채용 안내 문구나 복지 정보처럼 직무 매핑과 
직접 관련이 없는 표현은 제외하였다.
```
